# W1-1 retrospective — fix broken `fabric-lint --version`

W1-1 is the deliberately-small first unit shipped under dfg-harness
governance in tool-fabric. Single-line code change (Click decorator
argument) plus a regression test. The size is the point: the harness
chain around a 30-line patch is the demo.

The unit pivoted mid-cycle. The bootstrap PR (#1) originally proposed
"add `fabric-lint --version` flag" — which dual-critic caught as a
no-op (Click's `@click.version_option()` was already in place).
Empirical run of the existing flag revealed a real RuntimeError:
Click's package-name inference fails because the entry-point's module
is `linter.cli` but the installed distribution is `tool-fabric`. W1-1
was reframed (cycle-2 of PR #1) to fix that real bug.

This pivot itself validates the harness: the discipline caught a
weak unit selection BEFORE code was written, and the same chain
absorbed the new (better) unit definition without ceremony rework.
